# 产品决策文档
| 决策点 | 最终选择 | 判断依据 | 备选方案 | 潜在风险 |
| ---- | ---- | ---- | ---- | ---- |
| MVP 技术选型 | 规则引擎 | 快速验证策略闭环，可解释性强，便于演示 | 逻辑回归模型 | 规则颗粒度较粗 |
| 流失定义窗口 | 14天未登录 | 覆盖两个完整使用周期，误判率适中 | 7天 / 30天 | 不同用户群周期差异大 |
| 干预冷却期 | 7天 | 平衡挽留触达强度与用户反感度，参考行业实践 | 3天 / 14天 | 可能错过最佳挽留窗口 |
| 价值分层系数 | 1.5 / 1.0 / 0.5 | 优先挽回高 LTV 用户，资源效率最大化 | 2.0 / 1.0 / 0 | 系数偏主观，需真实 LTV 校准 |
| AI 能力实现方式 | 本地 Mock 数据 + 规则引擎 | 零外部依赖，演示稳定可靠，架构上预留真实 API 接口，可随时切换 | 直接调用外部大模型 API | Mock 数据覆盖场景有限，个性化程度依赖预置数据 |
| 功能体验用户处理 | 排除推送干预池，流转工单 | 文案无法解决产品缺陷，干预 ROI 为负 | 全部推送干预 | 漏判无投诉的体验问题用户 |
| 通用能力选型 | Pydantic / tenacity / statsmodels | 减少重复造轮子，降低 Bug 率，符合业界标准 | 全手写实现 | 增加少量依赖学习成本 |
